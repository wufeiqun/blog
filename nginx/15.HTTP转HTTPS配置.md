#### 背景介绍

经常会用到配置HTTP转HTTPS的时候, 记录一下.


#### 配置

# HTTP 转发到 HTTPS, 状态码使用307是因为大厂都在使用, 兼容性更好之类的原因
```
server {
	listen 80;
	server_name www.baidu.com;
	return 307 https://$host$request_uri;
}
```

#### 问题记录

2026-06-03:

公司有同事反馈使用HTTP访问和HTTPS访问, 相同参数, 但是返回不同, 后来经过排查结论如下:

```
客户端请求, NGINX返回301跳转. ，客户端（浏览器、Postman、Axios、Curl 等）在处理 301 重定向时，会自动将 POST 请求的方法修改为 GET，并且会丢弃原本的 POST Body（请求体数据），然后再向新的目标地址发起请求:

HTTP POST → 301 redirect → curl 自动变成 GET → 无 body → 默认参数 → 结果异常
```

NGINX的配置如下:
```
  server {
    server_name *.xxx.com;
    listen 80;
    if ($scheme = http) {
      return 301 https://$host:443$request_uri;
    }

    ...省略...

  }
```

通过curl的详细日志可以验证:
```
curl -vL --location 'http://xxx.com.cn/api/xxx' \

...省略...

< HTTP/1.1 301 Moved Permanently
< Location: https://xxx.com.cn:443/api/xxx

...省略...

* Switch from POST to GET

...省略...

* using HTTP/1.x
> GET /api/xxx HTTP/1.1

...省略...

```

结论:

> 跳转要使用307或者308, 这个是用来替换302和301的. 会强制客户端继续使用 POST 方法并携带原 Body 访问新地址.
