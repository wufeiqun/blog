#### 查看半连接配置和使用情况



>  结论: 半连接队列一般设置稍微大点后一般不会出问题, 设置比如4096就可以, 主要的问题主要是全连接!



```shell
# 查看当前系统TCP半连接队列大小, 一般服务器设置4096就足够了
# sysctl net.ipv4.tcp_max_syn_backlog
net.ipv4.tcp_max_syn_backlog = 4096

# TCP半连接队列满的时候的默认处理策略
# 开启syncookies后队列满了会把溢出的连接加上一定的算法加一个cookie, 客户端会重连, 重连后可以解密
# 作用是过滤那些SYN flood攻击的请求, 也就是队列满了的时候, 正常请求会重试兜底的
# 这样即使队列溢出，合法连接依然可以建立。默认值是1, 就是队列溢出的时候开启, 平常没啥影响
# sysctl net.ipv4.tcp_syncookies
net.ipv4.tcp_syncookies = 1

# 溢出的连接会被计入如下的计数器中
# 该方式查询的是一个递增的计数器, 需要观察变化
# 即使溢出也不表示该连接就失败了, 所以这里只是观察队列溢出情况
# netstat -s | grep -i "SYNs to LISTEN"
    35 SYNs to LISTEN sockets dropped

# 上述命令的统计信息也可以通过prometheus查询, 这样就可以查看图表, 看趋势
node_netstat_TcpExt_ListenDrops{}
```



#### 查看全连接配置和使用情况



```shell
# 查看当前的全连接的配置, 如果可以, 也要看看下游服务的连接数情况, 比如tomcat等
# sysctl net.core.somaxconn
net.core.somaxconn = 65535

# 查看全连接队列满的时候溢出连接的处理方式
# 0, 表示静默丢包, 客户端的现象是连接超时(connect timeout)
# 1, 给客户端发送一个RST, 客户端的现象是连接拒绝(connection refused)
# sysctl net.ipv4.tcp_abort_on_overflow
net.ipv4.tcp_abort_on_overflow = 0

# (实时查看)查看TCP全连接队列满后溢出的连接
# Recv-Q (Receive Queue): 当前队列里面的排队数量
# Send-Q (Send Queue)：全连接队列的最大长度
((myenv) ) [root@vps conf.d]# ss -lnt
State       Recv-Q      Send-Q           Local Address:Port             Peer Address:Port
LISTEN      0           65535                  0.0.0.0:80                    0.0.0.0:*
LISTEN      0           128                    0.0.0.0:22                    0.0.0.0:*
LISTEN      0           65535                  0.0.0.0:443                   0.0.0.0:*

# (查看历史)查看TCP全连接队列满后溢出的连接
# 这是一个累增的计数器
# netstat -s | grep -i "listen"
47 times the listen queue of a socket overflowed

# 上述监控指标也可以通过prometheus查看, 可以查看图表更加形象
node_netstat_TcpExt_ListenOverflows{}


```



#### 查看后端应用的QPS

一般客户端发生连接超时的时候会伴随着并发的增加, 可以查看后端的应用或者网关的并发QPS, 公司的这次问题的情况是默认的队列大小是128, 所以QPS150以及以上的时候发生了.



#### 修改内核参数的方法



```shell
# 临时生效
sysctl -w net.ipv4.tcp_max_syn_backlog=4096
sysctl -w net.core.somaxconn=4096

# 永久生效
echo "net.ipv4.tcp_max_syn_backlog = 4096" >> /etc/sysctl.conf
echo "net.core.somaxconn = 4096" >> /etc/sysctl.conf
sysctl -p
```

#### 修改NGINX参数方式

在`listen`后面加上backlog, 不用每个域名都加, 80改一个443的改一个就可以.

```
server {
    listen 443 ssl http2 backlog=65535;
    server_name wufeiqun.top www.wufeiqun.top;

    ssl_certificate     /etc/nginx/cert/wufeiqun.top.pem;
    ssl_certificate_key /etc/nginx/cert/wufeiqun.top.key;

    location / {
	    default_type text/plain;
	    return 200 "OK";
    }
}

server {
    listen 80 backlog=65535;
    server_name wufeiqun.top www.wufeiqun.top;
     return 301 https://$host$request_uri;

}
```
