使用场景:

主从模式下, 如果不希望从库写入数据的时候.

临时:

SET GLOBAL read_only = ON;

SET GLOBAL read_only = OFF;

SHOW GLOBAL VARIABLES like 'read_only';

持久化的话:

read_only = ON



1. read_only
  1.1 限制普通用户写操作
  1.2 拥有 SUPER 权限的用户可写
  1.3 防止误写
2. super_read_only
  2.1 限制所有用户（包括 SUPER）写操作
  2.2 复制线程除外
  2.3 只读从库保护

以上两个配置都不影响主从同步线程的写入.
