## 背景介绍

在重做MySQL主备的时候, 经常出现如下的问题:

```
# mysql --max_allowed_packet=1G  --default-character-set=utf8mb4 -h 192.168.1.1 -uroot  -p < data.sql
Enter password:
ERROR 1153 (08S01) at line 32342: Got a packet bigger than 'max_allowed_packet' bytes
```

客户端命名已经加上`max_allowed_packet`参数了, 依然还会报错. 这个报错的原因是不只是客户端要加上该参数, MySQL服务端也需要调整该参数.

## 解决方式

* 查看MySQL当前的配置

```sql
SHOW VARIABLES LIKE 'max_allowed_packet';
```

> MySQL5.7 默认是4M, MySQL8默认是64M.


* 临时调整

> 1GB,  1024 * 1024 * 1024=1073741824
>
> 64M , 64 * 1024 * 1024=67108864

```sql
SET GLOBAL max_allowed_packet=1073741824;
```

重新连接后生效.

* 永久修改

> 永久配置不建议设置为1GB, 因为会占用内存, 有潜在的OOM风险, 一般64M就足够了, 如果太大了, 那就得检查下SQL设计问题了.

```
[mysqld]
max_allowed_packet=67108864
```
