## 背景介绍

有时候binlog太多会占用大量的磁盘空间, 需要手工清除一部分.

## 执行过程

* 关注Slave从库当前正在读取的主库的binlog文件

删除binlog之前一般需要关注该实例的从库正在读取的binlog文件, 不然删除后主从就会出问题.

在从库上执行如下命令:

```
mysql> show slave status\G;
*************************** 1. row ***************************
               Slave_IO_State: Waiting for source to send event
                  Master_Host: wfq-mysql8-test-8fbfe-master.wufeiqun-test
                  Master_User: root
                  Master_Port: 3306
                Connect_Retry: 60
              Master_Log_File: mysql-bin.000005
          Read_Master_Log_Pos: 197
               Relay_Log_File: wfq-mysql8-test-8fbfe-salve-0-relay-bin.000010
                Relay_Log_Pos: 326
        Relay_Master_Log_File: mysql-bin.000005
             Slave_IO_Running: Yes
            Slave_SQL_Running: Yes
```

其中`Master_Log_File`是从库当前正在读取的主库文件.


* 删除指定文件之前的binlog文件

```
-- 从库在读 000005，那么 000004 及之前的都可以删
-- 其实下面的命令不会删除mysql-bin.000004, 保险期间还是少删除一个
PURGE BINARY LOGS TO 'mysql-bin.000004';

-- 查看清理后剩余
SHOW BINARY LOGS;
```
