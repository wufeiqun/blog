## 背景介绍

工作中经常遇到容器中网络问题, 比如网络不通, DNS解析异常等等. 即使可以通过shell进入网络, 也没有必要的工具, 还得临时安装, 非常麻烦. 通过nsenter可以在该容器所在的宿主机上直接进入容器的网络, 并且可以使用宿主机的网络工具, 非常方便.

## 使用方式

* 找到容器的进程ID

```
docker ps -a|grep xxx

docker inspect <container ID> |grep Pid
Pid: xxx

```

* 进入PID的网络命名空间

```
nsenter -n -t <PID>
```

* 自由发挥了

退出的话执行`exit`即可.
